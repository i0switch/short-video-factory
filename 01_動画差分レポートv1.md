# 動画差分レポート：理想参照 vs 現在実装

## A. 総評

**再現度：約30〜35%（致命的に乖離している）**

根本的な構造差が3つある：
1. **ランキング数が10→5に半減** — 参照は第10位〜第1位の10段構成、現在は第5位〜第1位の5段のみ
2. **コメント2段構成（ツッコミ演出）が完全欠落** — 参照では各順位に「コメント＋ツッコミ」の2段ボックス表示があるが、現在はタイトル1行＋画像のみ
3. **素材画像がイラスト→写真に変化** — 参照は「いらすとや」風フラットイラスト、現在はストック写真＋Fallback表示あり

テンプレートとして「同じフォーマット」と言えるレベルにするには、構造ロジック・レイアウトパターン・タイミング設計の3層すべてを修正する必要がある。

### C-2. レイアウトパターンの欠落

**参照動画には2種類のシーンパターンがある：**

#### パターンA：タイトル表示（各順位の冒頭）
```
┌─────────────────────┐
│     第N位            │ ← 画面上部 15〜25%
│                      │
│  ★タイトルテキスト★  │ ← 大きなテキスト（縁取り）画面中央〜やや上
│                      │
│    [イラスト画像]     │ ← 画面下部 50〜75%
│                      │
└─────────────────────┘
```

#### パターンB：コメント表示（各順位の展開）
```
┌─────────────────────┐
│     第N位            │ ← 画面上部 15〜25%
│                      │
│ ┌─青枠ボックス────┐  │ ← コメント1（白背景・青太枠）
│ │ コメントテキスト │  │
│ └─────────────────┘  │
│ ┌─赤枠ボックス────┐  │ ← コメント2 / ツッコミ（白背景・赤太枠）
│ │ ツッコミテキスト │  │
│ └─────────────────┘  │
│    [イラスト画像]     │ ← 画面下部
└─────────────────────┘
```

**現在出力はパターンBが完全に存在しない。** 全シーンがパターンAの変形（順位＋1行テキスト＋画像）のみ。

**参照の典型的な1順位のフロー（約5〜6秒）：**
```

0.3s  タイトルテキスト ポップイン
0.5s  イラスト フェードイン
2.5s  （タイトル表示フェーズ終了）
```

## D. 修正優先度つき差分一覧

### HIGH — これを直さないとテンプレートとして成立しない

| # | 問題 | 理想との差 | 修正方針 | 実装ヒント |
|---|------|----------|---------|-----------|
| H1 | **コメント2段表示パターンが存在しない** | 参照では各順位に「コメント＋ツッコミ」の2段ボックスが必ず表示される。現在は1段のみ | シーン構成に「コメントフェーズ」を追加。1順位 = タイトルフェーズ(2.5s) → コメントフェーズ(3s) の2段階にする | コンポーネント: `CommentPhase`。青枠Box + 赤枠Box を上下に配置、順次スライドインアニメーション |
| H2 | **1順位あたりの表示時間が2〜3倍長い** | 参照=約5s/順位、現在=10〜15s/順位 | 各シーンの固定表示時間を短縮。タイトル2.5s + コメント3s = 計5〜6s/順位に調整 | `sceneDuration` パラメータを順位ごとに設定。トランジション含め最大6秒 |
| H3 | **ランキング数が5段しかない** | 参照は10段（第10位〜第1位） | テンプレートとして10段対応にする。データ配列を10件分用意 | `rankingItems[]` 配列を10件に拡張。各要素に `title`, `comment1`, `comment2`, `image` を持たせる |
