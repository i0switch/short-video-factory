# 動画差分レポート v2

## A. 総評

比較の結論だけ先に言うと、総尺とランキングの大枠は合ってるけど、1シーン内の情報密度・要素の出し順・テキスト演出・素材表示がかなり別物になってる。
いちばん効いてる差分はこの4つ。

- 素材画像がほぼ出ていない / fallback が見えてる
- メイン文言の見た目が弱すぎて、参照の"刺さる圧"が出てない
- シーン内の時間配分が違い、空白時間が多い
- 場面切替で黒っぽいディップが入り、参照よりテンポが鈍い

なので、今の出力は「構成は似てるけど、1秒あたりの画面情報量が足りない」状態。
最短で寄せるなら、背景より先に **素材表示** と **文字演出** と **シーン内タイムライン** を直すべき。

補足:
- 参照動画は 360x640、実装出力は 1080x1920 だったので、位置とサイズは 1080x1920 換算の近似で書いてる
- 総尺はかなり近い
  - 参照: 59.584s
  - 実装: 59.562s
- シーン境界の近似比較:
  - 参照: 0.00 / 3.44 / 8.20 / 13.74 / 19.28 / 24.75 / 29.65 / 35.12 / 39.92 / 45.73 / 51.53 / 57.47 / 59.58
  - 実装: 0.00 / 3.40 / 8.80 / 14.20 / 19.60 / 25.00 / 30.40 / 35.80 / 41.20 / 46.60 / 52.00 / 57.40 / 59.56
- つまり、総尺は一致に近いのに、各シーンの中身が後ろ寄り・均等寄りになってる。
  参照はシーンごとに少し長短があるけど、実装はほぼ全部 5.4s固定。

## B. 一致している点

- 9:16 縦動画の前提は一致
- オレンジ系の放射背景を使う方向性は一致
- ランキング進行 10位→1位→CTA の構造は一致
- シーン数も実質 12ブロック（オープニング / 10位〜1位 / エンディング）
- 総尺もほぼ一致

ここは土台としてOK。
ただし、一致してるのはあくまで骨組みで、見た目の再現度はまだ低い。

## C. 大きくズレている点

### 素材画像レイヤー
- 参照はほぼ全シーンで、下部に素材画像が常時見えてる
- 実装は未表示か、灰色の fallback 四角が遅れて出る
- これだけで再現度が一気に落ちてる

### メイン文言の演出
- 参照は赤文字 + 白縁 + 強い黒シャドウ/黒ボリュームで中央に強く載る
- 実装は薄いピンクっぽい文字 + 弱い縁/影で背景に埋もれてる

### シーン内の出し順
- 参照は「切り替わった瞬間に情報が埋まる」
- 実装は「切替 → ちょい空白 → 小さい文字 → また空白 → 箱」
- 空白時間が目立つ

### 場面転換
- 参照はかなり切れ味のある切替
- 実装は 黒ディップ / 暗転気味の繋ぎが入る（例: 8.8〜9.0s 付近で黒が見える）

### 背景パターン
- 参照は単純な放射だけじゃなく、ジグザグ/山型の面構成が混ざった背景
- 実装は細い放射線が中心から均等に広がるだけ
- 背景の"厚み"が違う

### 画面密度
- 参照は常に「順位 / メイン文言 / 素材 / 後半はコメント箱」で埋まる
- 実装は同時表示要素数が少なく、スカスカに見える

## D. 修正優先度つき差分一覧

### 1. AssetImageLayer / 素材画像表示
**優先度: HIGH**

**問題**: 素材画像がほぼ出ていない。出ても灰色 placeholder が visible になっている。

**理想との差**: 参照では各順位シーンで素材画像が下部中央〜下部やや左に常時表示。実装では:
- オープニング: 画像なし
- 各ランキング: 画像なし or 遅れて灰色 fallback

**修正方針**:
- 画像ロード成功前に scene を進めない
- 画像未取得時は placeholder を見せない
- fallback は render 用ではなく debug 用に限定
- 各シーン開始直後から素材を出す

**実装ヒント**:
- delayRender() / continueRender() で素材ロード完了待ち
- Img の失敗時は null を返し、灰色 box を render しない
- 1080x1920換算の目安:
  - 素材画像枠: w 300〜380, h 300〜420
  - アンカー: centerX 540 ± 100
  - y 1320〜1710
  - scene開始から 0〜6f以内に表示開始

### 2. MainCaption / メイン文言
**優先度: HIGH**

**問題**: メイン文言が小さい・薄い・背景に埋もれる。

**理想との差**: 参照は中央に大きい赤文字が載り、白縁 + 強い黒シャドウで背景に勝ってる。実装は薄ピンク系で、太さもコントラストも足りない。

**修正方針**:
- メイン文言は別スタイルに固定
- 文字色は赤系、太め、白縁、黒シャドウ強め
- 横幅を大きくとり、2行以内で中央密度を作る

**実装ヒント**:
- 1080x1920換算:
  - テキストブロック x 90〜990, y 670〜1080
  - 幅 820〜930
- スタイル例:
  - fontWeight: 900
  - fill: 赤
  - stroke: 白 10〜14px
  - shadowBlur: 18〜28
  - shadowColor: rgba(0,0,0,0.85)
  - lineHeight: 0.95〜1.05
- 改行は短すぎない。1行あたりの長さを揃える

### 3. SceneSequencer / 1シーン内の進行
**優先度: HIGH**

**問題**: シーン内の情報が出る順番が参照と違う。空白時間が入って、密度が落ちてる。

**理想との差**:
参照は1シーン内でだいたいこう:
- Phase A: 切替直後に 順位 + メイン文言 + 素材
- Phase B: 後半で コメント箱1
- Phase C: 必要なら コメント箱2
- 次シーンへ即切替

実装はこうなってる:
- 切替 → 何もない時間 → 小さい本文 → また間 → コメント箱 → fallback

**修正方針**: シーンを固定フェーズ化する。

**推奨タイムライン**:
- 0.00〜0.15s: 順位出現
- 0.00〜1.80s: 順位 + メイン文言 + 素材を同時表示
- 1.80〜3.10s: コメント箱1
- 3.10〜4.40s: コメント箱2
- 4.40〜4.75s: 次シーンへ

### 4. TransitionLayer / シーン切替
**優先度: HIGH**

**問題**: 黒っぽい暗転が入ってテンポを落としてる。

**修正方針**:
- 全画面暗転を削る
- 切替は hard cut ベース
- Fade を使うなら 2〜4f 程度
- 背景全体を暗くしない
- 黒板を一枚被せる実装は外す

### 5. OpeningHero / オープニング
**優先度: HIGH**

**問題**: オープニングに素材画像がなく、タイトルも迫力不足。

**修正方針**:
- openingでも素材画像を常時表示
- タイトルサイズを増やす
- シャドウと縁取りを参照寄せ

**実装ヒント**:
- タイトル塊: x 40〜720, y 80〜600
- 下部画像: x 150〜430, y 1080〜1600
- opening durationは約 3.4s
- rank10への繋ぎは暗転なし

### 6. RankBadge / 順位表示
**優先度: MEDIUM**

**問題**: 順位表示が小さく、輪郭も弱い。

**修正方針**:
- 順位ラベルを 1.3〜1.5倍
- 黒シャドウを厚く
- 常時固定表示

**実装ヒント**: 1080x1920換算:
- centerX ≈ 540, y 180〜330
- 幅感 380〜520
- 白字 + 太黒影で固定

### 7. CommentBoxGroup / コメント箱
**優先度: MEDIUM**

**問題**: 箱が小さい・薄い・遅い・上に寄りすぎ。

**修正方針**:
- 箱を大型化
- 枠を太く
- 上段1個 + 中段1個の2段構成に寄せる
- 出現はスライド/ポップで一気に

**実装ヒント**: 1080x1920換算:
- 箱幅 700〜920
- 箱高 120〜170
- 1つ目 y 300〜470
- 2つ目 y 500〜700
- border: blue 8〜10px, red 8〜10px
- テキスト: 48〜60px, 黒太字
- imageは箱出現中も残す

### 8. BackgroundPattern / 背景
**優先度: MEDIUM**

**問題**: 背景のパターンが参照と違う。実装は"単なる放射線背景"になってる。

**修正方針**: 背景をテンプレート化し直す。

**実装ヒント**:
- 焦点をやや下へ: centerX ≈ 540, centerY ≈ 1040〜1120
- ray を太く
- chevron / zigzag パターンを重ねる
- sceneごとの差は最小、まずは静止背景でOK

### 9. CTAEnd / 最後のコメント誘導
**優先度: MEDIUM**

**問題**: 終盤CTAが小さく、参照の押し出しが出てない。

**修正方針**:
- CTA全体を大型化
- 赤強調行をもっと目立たせる
- 黒シャドウを厚く

**実装ヒント**:
- CTA block: x 80〜1000, y 1080〜1650
- 1行目白, 2行目赤強調 + 白縁 + 黒シャドウ

### 10. LayoutSpacing / 余白と密度
**優先度: MEDIUM**

**問題**: 縦方向の間が空きすぎて、参照より薄い。

**修正方針**: グリッドで固定する。

**実装ヒント**:
- 上 15%: 順位
- 中 35〜55%: メイン文言
- 下 72〜92%: 素材
- 後半 25〜55%: コメント箱

### 11. TextWrap / 改行
**優先度: LOW**

**問題**: 改行が短くなって横幅が使えてない。

**修正方針**:
- balanced wrap
- 1行の長さを揃える
- キャプション幅は 820〜930

### 12. StylingTokens / スタイル定義の分離
**優先度: LOW**

**問題**: 本文、箱、順位、CTA の役割差が弱い。

**修正方針**: 5トークンを先に固定:
- RankStyle
- MainClaimStyle
- CommentBoxStyle
- CTAStyle
- ImageStyle

## E. 実装タスク一覧

1. 素材画像ロード処理を修正（fallback の可視表示を止める、scene開始時から画像を見せる、preload 完了待ちを入れる）
2. メイン文言スタイルを作り直す（赤文字、白縁、強黒シャドウ、サイズ拡大、2行以内の整形）
3. 順位ラベルのサイズと影を強化（上部中央固定、scene中ずっと表示）
4. シーン内タイムラインを再設計（本文+画像先出し、箱は後半、空白区間ゼロ）
5. 切替暗転を除去（hard cut に寄せる、fade を超短尺化）
6. コメント箱のレイアウト再設計（大型化、太枠化、上段/中段の2段構成）
7. 背景テンプレート再作成（焦点位置修正、ray 太さ調整、chevron感追加）
8. オープニング再構築（タイトル拡大、素材画像追加、rank10への切れ味改善）
9. CTA再構築（最終文言拡大、赤強調をもっと強く、黒影を厚く）
10. フレーム基準のQAを導入（比較固定点: 1.5s / 4.7s / 10.0s / 31.2s / 34.4s / 58.0s）

## F. 最短で再現度を上げるための修正順序

1. **素材画像の表示を直す** — まずこれ。ここが死んでると何やっても参照に見えない。
2. **メイン文言の見た目を直す** — 赤/白/黒の強い文字演出に寄せる。
3. **シーン内タイムラインを直す** — 空白を消して、scene頭から情報を埋める。
4. **暗転トランジションを削る** — テンポを参照に近づける。
5. **コメント箱の大きさ・位置・出現順を直す** — 後半フェーズの密度を上げる。
6. **背景パターンを寄せる** — ここで土台の"質感"が揃う。
7. **オープニングとCTAを仕上げる** — 入口と出口の印象を揃える。
8. **フレーム比較で微調整** — 4.7s / 31.2s / 34.4s / 58.0s を固定検査点にする。

いちばん大事な一文で締めると、
今のズレは「背景」より「素材が出てない」「本文が弱い」「シーン内の出し順が違う」が支配的。
ここを直せば、再現度はかなり一気に上がる。
